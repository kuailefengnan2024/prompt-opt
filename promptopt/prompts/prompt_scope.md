# 提示词语义分层（与输入格式无关 · 必须先理解再改）

**输入可能是任意格式**：三段标题、两段、纯段落、中英混排、bullet 列表等。**不得**因为没有【核心特征】等标题就放弃分层或全文乱改。

## 第一步：自行识别三层（语义推断，非格式匹配）

| 层 | 识别线索（满足任一即可归入） | 能否改 |
|----|------------------------------|--------|
| **设计要求约束** | 上方「约束锚点」+ 主副标题、必须出现的意象/文案、业务调性要求 | **否** |
| **核心特征约束** | 主体是什么、符号/隐喻、创意内核、标题文案内容；宜简洁 | **否** |
| **画面风格约束** | 整体调性、材质体系、光影气质、商业感/高级感定调、全图色彩基调 | **否** |
| **可改层** | 画面**有哪些元素**、元素**位置/大小/层次/前后景**、**动线/留白**、标题**排版位置**、**为构图服务的局部色彩**（非全图风格定调） | **是** |

**混排时：** 同一段里可能既有约束又有构图——只改描述**空间关系与元素布局**的句子，不动定义「是什么 / 什么风格」的句子。

**无标题时：** 在全文搜索描述「元素 + 空间关系」的句子作为 `target`；不要把定义意象/风格/标题内容的句子当作 target。

## 第二步：可改层局部 patch（可归因 · 肉眼可辨）

- `target` = 可改层内**精确原文**（优先 2～5 句子块；必要时可覆盖整段【画面构图】正文，但不要连同约束层一起改）。
- `content` 相对 target 可达 **500%**；每条 edit 必须带来**出图后肉眼可见**的变化，例如：
  - 景别/视角切换（平视↔微仰、中景↔近景）
  - 主次落点迁移（第一视觉从 A 移到 B）
  - 元素增删或空间重排（前后景、左右、高低、疏密）
  - 动线/留白重构（引导路径、留白比例）
- 优先做**出图可见**的结构性调整（景别/落点/层次/动线/元素关系）。
- 约束层一字不动。审美问题若落在约束层，只能通过**构图布局**间接缓解。

## patch 中必须附带分层说明

在 `patch.layer_analysis` 写清：约束是什么、本次改的是哪个构图子块、预期出图差异是什么。
